# AI Writing Agent

一个功能完善的 AI 文稿工作流 Agent，支持读取多类型材料文件，自动完成资料汇总、关键信息提取、文稿初稿生成、逻辑评审、事实风险检查和最终润色，并输出 Markdown、JSON、TXT 等格式结果。

本项目适用于科研项目总结、申报材料、实践活动报告、新闻稿、论文段落修改、阶段成果汇报等场景，能够帮助用户减少重复性材料整理工作，提高文稿生成效率和内容规范性。

---

## 一、项目简介

在科研、项目申报、活动总结和论文写作过程中，常见的问题包括：

- 材料来源分散，需要人工反复整理；
- 多份文档之间表述不一致；
- 初稿撰写耗时较长；
- 文稿逻辑链条不完整；
- 成果表述容易空泛或缺乏事实依据；
- 修改时难以及时发现夸大、不严谨或前后矛盾的问题。

AI Writing Agent 通过多阶段 AI 工作流，将“资料读取—信息提取—初稿生成—逻辑评审—事实风险检查—最终润色”拆分为多个独立环节，使文稿生成过程更加稳定、清晰、可控。

---

## 二、核心功能

### 1. 多类型材料读取

支持读取常见文本材料，包括：

- `.txt`
- `.md`
- `.docx`
- `.pdf`

系统会自动扫描输入文件夹中的材料，并将其整理为统一的文本输入。

---

### 2. 材料分块处理

当输入材料较长时，Agent 会自动进行文本分块，避免一次性输入内容过长导致模型处理效果下降。

分块处理适用于：

- 长篇会议记录；
- 多份项目材料；
- 长篇论文摘要；
- 大量活动记录；
- 多阶段工作总结。

---

### 3. 关键信息提取

Agent 会从原始材料中提取结构化信息，包括：

- 项目或任务背景；
- 要解决的核心痛点；
- 已有工作基础；
- 技术路线或工作方法；
- 阶段性成果；
- 可量化数据；
- 存在的问题；
- 后续计划。

在材料没有明确说明的情况下，系统会标注“材料中未明确说明”，避免自动编造事实。

---

### 4. 文稿初稿生成

系统可根据提取出的关键信息，生成不同类型的文稿初稿，例如：

- 项目成果总结；
- 申报书段落；
- 实践活动报告；
- 新闻稿；
- 论文相关段落；
- 阶段性工作汇报。

文稿生成时会尽量保持逻辑清晰、表达稳重，并避免空泛套话。

---

### 5. 逻辑评审

Agent 会对初稿进行逻辑检查，重点判断：

- 核心痛点是否清楚；
- 技术路线是否具体；
- 成果表述是否有依据；
- 方法和结果是否对应；
- 是否存在前后矛盾；
- 是否存在夸大或不可验证的说法；
- 哪些地方需要补充事实或数据支撑。

---

### 6. 事实风险检查

系统会识别文稿中可能存在风险的表达，例如：

- 没有材料依据的数据；
- 过度绝对化的表述；
- 夸大成果的说法；
- 模糊但看似确定的结论；
- 难以核查的影响范围；
- 缺少证据支撑的效率提升表述。

该功能可以帮助用户在提交材料前提前发现潜在问题。

---

### 7. 最终润色

在完成逻辑评审和事实风险检查后，Agent 会根据评审意见对文稿进行修改，生成最终版本。

最终版本会尽量做到：

- 保留真实信息；
- 弱化缺乏依据的表述；
- 增强段落衔接；
- 提高语言规范性；
- 保持自然、稳重的中文表达；
- 不新增材料中没有的事实。

---

### 8. 多格式输出

支持输出多种结果文件：

- `Markdown`：适合人工阅读和继续修改；
- `JSON`：适合程序化处理或二次开发；
- `TXT`：适合纯文本保存和快速复制。
